# Changelog

## 0.1.2-beta (Feature Expansion & House Ads Engine)

- **Complete AdMob Removal & Self-Hosted House Ads Engine**: Stripped all third-party tracking and ad SDKs. Transitioned 100% of promotional delivery to self-hosted, privacy-friendly **House Ads** fetched via `HouseAdRepository` (`https://house.monkey-network.xyz/api/public/ads`).
- **Dynamic Interspersed & Summary Page Promotions**: Banners dynamically intersperse into the news stream (`NewsList`) after every 4th story (`itemCount % 4 == 0`), cycling cleanly through all active campaigns. Embedded `HouseAdBanner` directly on the individual article summary view (`StoryDetail`) using deterministic article hashing (`(item.id.hashCode() and 0x7FFFFFFF) % activeAds.size`). Replaced "Sponsored" terminology with clean, native `"Promo"` and `"Tap to learn more"` badges.
- **Offline Bookmarks & "Saved Stories" Section**: Added dedicated **Saved Stories (`SAVED`)** section in the main section menu with an icon indicator. Expanded `BudgieArticleDatabase` SQLite schema with a `bookmarked_articles` table and methods (`toggleBookmark`, `isBookmarked`, `bookmarkedIds`, `allBookmarks`) allowing users to bookmark any story across `LeadStory`, `StoryCard`, and `StoryDetail` for reading anywhere, even offline.
- **Dedicated "General News" (`GENERAL`) Category**: Added a new main section tab (`General`) right alongside `Headlines`, `Breaking`, and `Important` across all 4 outlets (BBC, Sky, Guardian, Sun). Intelligently filters out urgent breaking alerts to give users clean, everyday general reporting, lifestyle, and culture news.
- **Hybrid Neural Cloud & On-Device TTS Audio Reader with Instant `<1.2s` Playback**: Upgraded `BudgieAudioReader` to a hybrid audio streaming engine optimized for instant starting (<1.2s delay). When online, it connects to a 100% free, unauthenticated public voice API (`StreamElements/Amazon Polly`) streaming the ultra-natural `Brian` (British English Male News Anchor) neural voice directly to `MediaPlayer` (`1.0x -> 1.25x -> 1.5x`). If network voice preparation exceeds a strict 1.2-second Handler timeout or when offline, it instantly switches to high-quality on-device neural `TextToSpeech` with sentence-by-sentence queueing (`QUEUE_FLUSH -> QUEUE_ADD`) so playback begins out loud in `<50ms`.
- **Instant Keyword Search**: Added an interactive search toggle in the Top Bar (`Icons.Rounded.Search`). Users can instantly filter loaded news stories by keyword or topic matching titles, descriptions, and source outlets.
- **House Ads Engine Registration Polish**: Aligned `REGISTERED_PACKAGE_NAME` in `HouseAdRepository` to exact `applicationId` (`com.budgienews.app`), permanently eliminating `BuildConfig.APPLICATION_ID mismatch` warning flags when requesting banners from the house ads network (`https://house.monkey-network.xyz/api/public/ads`).
- **Bulletproof Audio Reader & TTS Lifecycle Management**: Eliminated `speak failed: not bound to TTS engine` system warnings and audio drops in `BudgieAudioReader` by introducing explicit service binding awareness (`isTtsReady`) and an automatic request queue (`pendingTtsAction`). When speech playback is requested while Android's background TTS service is still initializing (`onInit`), audio requests are queued and dispatched immediately the exact millisecond the engine confirms successful binding (`status == TextToSpeech.SUCCESS`). Added automatic engine shutdown and re-initialization (`handleSpeakError`) if `tts.speak()` returns an error or encounters an unbound/crashed state during runtime.
- **Cloud Firestore Native Mode & Real-Time Sync Stability**: Transitioned `moneybytes-apk` database (`default`) from Datastore Mode to **Firestore in Native mode** (`europe-west2`). Resolved real-time `WatchStream` and `WriteStream` `FAILED_PRECONDITION` connection loops and main thread layout stuttering (`Choreographer Skipped frames`), ensuring clean out-of-the-box synchronization for live news (`articles` collection) and device notification tokens (`deviceTokens` collection).
